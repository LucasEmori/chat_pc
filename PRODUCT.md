# Product

## Register

product

## Users

Operadores de compras (ALAN, LUIS, etc.) que processam invoices de fornecedores chineses em pedidos de compra para ERP. Contexto: escritório, workflow repetitivo, precisão crítica em dados tabulares.

## Product Purpose

Converter invoices (.xlsx/.csv) em Pedidos de Compra formatados para ERP usando LLM com HITL (human-in-the-loop) para dúvidas. Chat conversacional como interface principal — upload, processamento, correções e download acontecem dentro de um fluxo de mensagens.

## Brand Personality

Preciso, minimal, sem ruído. Preto e branco. Nome: AzimeAI. Referência visual: Claude.ai. Ferramenta que desaparece no workflow.

## Anti-references

Decorativo, colorido, emojis excessivos, card grids, sidebar confusa, UI " enterprise dashboard". Nada de gradientes, glassmorphism, ou backgrounds decorativos.

## Design Principles

1. Chat-first: toda interação é uma mensagem. O usuário nunca "navega".
2. Monocromático: preto (#0a0a0a) e branco (#fafafa). Cinzas neutros como escala intermediária.
3. Densidade controlada: tabela de dados aparece como preview inline, não como página separada.
4. Feedback em tempo real: progresso, erros e HITL são mensagens no chat, não modais.
5. Zero装饰: sem ornamento. O conteúdo é a interface.

## Accessibility & Inclusion

WCAG 2.1 AA. Contraste >= 4.5:1 em todo texto. Keyboard navigation completo. Labels associados. Sem hover-only.
